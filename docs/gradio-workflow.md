# Gradio 双模式页面工作流

## 页面目标

页面需要同时承载两条流程：

- 关键词模式
- 论文库偏好模式

所以页面不是传统单输入框搜索页，而是一个“工作台”。

## 页面组成

- 模式切换
- 关键词输入框
- 开始检索按钮
- 刷新当天论文按钮
- 当前步骤显示
- 阶段日志显示
- 推荐结果卡片区

## 关键词模式

用户行为：

1. 选择关键词模式
2. 输入 query
3. 点击开始检索

后台流程：

1. 检查当天论文缓存
2. 构建当天论文索引
3. 检索当天论文
4. 生成命中论文中文总结
5. 写入结果 Markdown
6. 展示结果卡片

## 论文库偏好模式

用户行为：

1. 选择论文库偏好模式
2. 点击开始检索

后台流程：

1. 扫描 `paper_datasets`
2. 解析 PDF 前几页
3. 生成单篇 PDF 偏好总结
4. 汇总整体研究偏好
5. 检查当天论文缓存
6. 构建当天论文索引
7. 用偏好 query 检索当天论文
8. 生成命中论文中文总结
9. 写入结果 Markdown
10. 展示结果卡片

## 为什么要有阶段日志

因为偏好模式包含多步模型调用和本地缓存判断。

如果没有阶段日志，用户会不知道系统当前是在：

- 读 PDF
- 生成偏好画像
- 还是已经进入当天论文检索

阶段日志的价值不仅是 UX，更是调试能力。
