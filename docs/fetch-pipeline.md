# 最新一天论文抓取与缓存流程

## 目标

工程启动推荐流程时，不是每次都重新抓 arXiv，而是优先复用当天缓存。

## 规则

1. 先看 `output/daily/cs_cv_latest.md` 是否存在
2. 如果存在，并且 `date_slug` 是今天
   - 直接复用
   - 若有缺失摘要，再补英文摘要
3. 如果不存在，或日期不是今天
   - 重新抓取 `cs.CV/recent`
   - 落盘到 `latest` 和按日期归档文件

## 抓取字段

每篇论文至少包含：

- 标题
- PDF 链接
- HTML 链接
- Abstract 页面链接
- 英文摘要

## 为什么要先缓存再推荐

因为推荐模式有两条，但它们都依赖同一份“当天论文基础数据”。

把抓取和推荐拆开后：

- 关键词模式可以直接复用当天缓存
- 偏好模式也可以复用当天缓存
- 不会因为 UI 切换模式而重复请求 arXiv
